# Shuffle Workflow Development

## Objective

Develop an automated SOAR workflow in Shuffle to receive alerts from Wazuh, enrich them using VirusTotal, and automatically create incident cases in TheHive for SOC analysts.

---

## Workflow Overview

The workflow automates the complete incident handling process from detection to case creation.

```text
Windows Endpoint
        │
        ▼
Wazuh Agent
        │
        ▼
Wazuh Manager
        │
        ▼
Webhook Trigger (Shuffle)
        │
        ▼
Extract SHA-256 Hash
        │
        ▼
VirusTotal Reputation Lookup
        │
        ▼
Create Case in TheHive
        │
        ▼
SOC Analyst Investigation
```

---

## Workflow Development

### Step 1 – Create a Workflow

A new workflow was created in Shuffle Cloud to automate the incident response process.

---

### Step 2 – Configure the Webhook Trigger

A **Webhook** trigger node was added to the workflow.

Shuffle generated a unique webhook URL that was later configured inside the Wazuh Manager (`ossec.conf`) to automatically receive alerts generated by Rule **100002**.

---

### Step 3 – Configure VirusTotal

A VirusTotal node was added to the workflow.

Configuration included:

- Creating a VirusTotal account.
- Generating a personal API Key.
- Adding the API Key to the VirusTotal node for authentication.

The extracted SHA-256 hash from the Wazuh alert was supplied as the lookup value.

VirusTotal returned:

- File reputation
- Detection ratio
- File metadata
- Security vendor detections

---

### Step 4 – Configure TheHive

A dedicated user was created in TheHive.

An API Key was generated for that user.

Inside Shuffle:

- A TheHive node was added.
- TheHive server URL was configured.
- The user's API Key was added to authenticate the connection.

This allowed Shuffle to create cases directly inside TheHive.

---

### Step 5 – Extract Alert Information

The workflow extracted important information from the Wazuh alert, including:

- Rule ID
- Alert title
- SHA-256 hash
- File information

The extracted SHA-256 hash was then passed to the VirusTotal node.

---

### Step 6 – Create an Incident

After VirusTotal enrichment completed successfully, Shuffle automatically created a new case inside TheHive containing:

- Alert title
- Rule ID (100002)
- SHA-256 hash
- VirusTotal reputation
- Alert details from Wazuh

---

## Validation

The workflow was validated by intentionally executing **Mimikatz** on the Windows endpoint.

The following sequence was successfully verified:

- Wazuh detected the attack.
- Rule **100002** was triggered.
- Wazuh forwarded the alert to Shuffle using the webhook.
- Shuffle extracted the SHA-256 hash.
- VirusTotal returned the file reputation.
- Shuffle authenticated with TheHive using the API Key.
- A new incident was automatically created in TheHive.

---

## Result

A fully automated SOAR workflow was successfully developed. The workflow receives alerts from Wazuh, enriches them using VirusTotal, and automatically creates investigation-ready cases in TheHive, significantly reducing manual effort for SOC analysts.
